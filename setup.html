1️⃣ สมัครบัญชี Supabase

  1. เปิดเว็บไซต์ Supabase ไปที่ supabase.com แล้วคลิกปุ่ม "Start your project" หรือ "Sign Up"
  2. เลือกวิธีสมัคร — แนะนำสมัครด้วย GitHub (ง่ายที่สุด) หรือใช้ Email ธรรมดาก็ได้
  3. ยืนยันอีเมล — ถ้าสมัครด้วย Email จะมีลิงก์ยืนยันส่งมาให้ กดยืนยันก่อนเข้าใช้งาน

2️⃣ สร้าง Project ใหม่

  1. คลิก "New Project" — หลัง Login แล้วจะเห็น Dashboard คลิกปุ่ม "New Project"
  2. กรอกชื่อ Project — ตั้งชื่อตามชื่อหอพักของคุณ เช่น my-apartment
  3. ตั้งรหัสผ่านฐานข้อมูล — กรอก Database Password (เก็บไว้ด้วย แต่ระบบนี้ไม่ได้ใช้โดยตรง)
  4. เลือก Region — เลือก Southeast Asia (Singapore) เพื่อให้โหลดเร็วในไทย
  5. คลิก "Create new project" — รอประมาณ 1-2 นาที ระบบจะสร้าง Project ให้อัตโนมัติ
⚠️ ข้อควรระวัง:
  • Free Tier ของ Supabase สร้างได้ 2 Projects ต่อ 1 บัญชี
  • อย่าสับสน: "Database Password" ที่ตั้งในขั้นตอนนี้ ไม่ใช่ รหัสผ่านที่ใช้ Login เข้าเว็บออกบิล รหัสผ่านสำหรับ Login จะตั้งทีหลังในขั้นตอนที่ 5

3️⃣ สร้างตารางเก็บข้อมูล (2 ตาราง)

ไปที่เมนู SQL Editor ทางซ้ายมือ แล้วรัน SQL ด้านล่างนี้ ทีละส่วน

📋 คำแนะนำ: คัดลอก SQL แต่ละส่วน วางในช่อง SQL Editor แล้วกดปุ่ม "Run" (หรือ Ctrl+Enter) ทำทีละส่วนจนครบ

📋 ส่วนที่ 1: สร้างตาราง room_records

ตารางนี้เก็บประวัติบิลทั้งหมดที่เคยออก

-- ตารางที่ 1: เก็บประวัติบิลทั้งหมด
CREATE TABLE room_records (
    id             bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    timestamp      timestamptz DEFAULT now(),
    month          text,
    room           text,
    rent           numeric DEFAULT 0,
    water_before   real DEFAULT 0,
    water_after    real DEFAULT 0,
    water_unit     numeric DEFAULT 0,
    water_total    numeric DEFAULT 0,
    elec_before    real DEFAULT 0,
    elec_after     real DEFAULT 0,
    elec_unit      numeric DEFAULT 0,
    elec_total     numeric DEFAULT 0,
    internet       numeric DEFAULT 0,
    discount       numeric DEFAULT 0,
    grand_total    numeric DEFAULT 0
);

📋 ส่วนที่ 2: สร้างตาราง rooms_data

ตารางนี้เก็บข้อมูลห้องและผู้เช่าปัจจุบัน (ข้อมูลล่าสุดของแต่ละห้อง)

-- ตารางที่ 2: เก็บข้อมูลห้องและผู้เช่า
CREATE TABLE rooms_data (

    room_number   integer NOT NULL UNIQUE,

    fine          real DEFAULT 0,
    paid_status   boolean DEFAULT false,
    last_total    real,
    last_month    text,
    last_water    real,
    last_elec     real

4️⃣ ตั้งค่า Row Level Security (RLS)

RLS คือระบบป้องกันข้อมูล ทำให้เฉพาะผู้ที่ Login แล้วเท่านั้นที่เข้าถึงข้อมูลได้ รันใน SQL Editor ต่อจากขั้นตอนที่ 3

-- ========================================
-- เปิดใช้งาน RLS ทั้ง 2 ตาราง
-- ========================================
ALTER TABLE room_records ENABLE ROW LEVEL SECURITY;
ALTER TABLE rooms_data   ENABLE ROW LEVEL SECURITY;

-- ========================================
-- สร้าง Policy อนุญาตเฉพาะผู้ที่ Login แล้ว
-- ========================================
CREATE POLICY "auth_users_only_records"
ON room_records
FOR ALL
USING (auth.role() = 'authenticated');

CREATE POLICY "auth_users_only_rooms"
ON rooms_data
FOR ALL
USING (auth.role() = 'authenticated');
🔒 สำคัญมาก: ถ้าไม่รัน SQL ส่วนนี้ ข้อมูลในฐานข้อมูลจะเป็น Public ทุกคนสามารถอ่านและแก้ไขได้ กรุณารันทุกครั้ง

5️⃣ สร้างบัญชีผู้ใช้สำหรับ Login

บัญชีนี้คือ Email/Password ที่จะใช้ Login เข้าเว็บออกบิล (ไม่เกี่ยวกับบัญชี Supabase ที่สมัครไว้ในขั้นตอนที่ 1)

  1. ไปที่เมนู Authentication — คลิกที่ไอคอน 🔑 Authentication ในเมนูซ้ายมือ
  2. คลิก "Users" → "Add user" → "Create new user"
  3. กรอก Email และ Password — ใส่อีเมลและรหัสผ่านที่ต้องการใช้ Login เข้าเว็บออกบิล (จดไว้กันลืม)
  4. เปิด "Auto Confirm User" ✅ — เปิดสวิตช์นี้เพื่อให้ Login ได้ทันทีโดยไม่ต้องกดยืนยันอีเมล
  5. คลิก "Create user"
👥 หลายคนช่วยกันออกบิล? ถ้ามีหลายคนช่วยกันออกบิล สามารถทำซ้ำขั้นตอนนี้เพื่อสร้างหลายบัญชีได้ ทุกคนจะเห็นข้อมูลห้องเดียวกันหมด

6️⃣ ดึง API URL และ Anon Key

ค่า 2 อย่างนี้คือ "กุญแจ" ที่ใช้เชื่อมเว็บออกบิลเข้ากับฐานข้อมูลของคุณ

⚠️ Supabase ได้ย้ายตำแหน่ง: API URL และ Anon Key อยู่คนละหน้ากัน ต้องเข้าไปคนละเมนู อ่านให้ครบก่อนทำ

📡 6A — หา API URL

เข้าไปตามเส้นทางนี้:
⚙️ Project Settings › Integrations › Data API

ในหน้า Data API จะเห็นหัวข้อ "Project URL" คัดลอกค่าที่ขึ้นต้นด้วย https:// และปิดท้ายด้วย .supabase.co

🚫 สำคัญมาก — ใช้เฉพาะ URL หลักเท่านั้น!
  • ✅ ถูกต้อง: https://xxxxxxxxxxxxxxxx.supabase.co
  • ❌ ผิด: https://xxxxxxxxxxxxxxxx.supabase.co/rest/v1/
  • ❌ ผิด: https://xxxxxxxxxxxxxxxx.supabase.co/ (มี / ปิดท้าย)
ต้องตัด /rest/v1/ ออก และห้ามมี / ปิดท้าย

🔑 6B — หา Anon Key

เข้าไปตามเส้นทางนี้:
⚙️ Project Settings › API Keys › แท็บ "Legacy anon, service_role API keys"

ในแท็บ "Legacy anon, service_role API keys" จะเห็นคีย์ 2 ตัว ให้คัดลอกเฉพาะคีย์ที่ชื่อ "anon" (public) เท่านั้น เป็นค่าตัวยาวที่ขึ้นต้นด้วย eyJ...

🚫 ห้ามใช้คีย์ที่ชื่อ "service_role" เด็ดขาด! คีย์นี้สามารถข้าม RLS และเข้าถึงข้อมูลทั้งหมดได้ ให้ใช้แค่คีย์ "anon" เท่านั้น
✅ สรุปค่าที่ต้องเก็บไว้:
  • API URL → มาจาก Project Settings › Integrations › Data API
  • Anon Key → มาจาก Project Settings › API Keys › แท็บ "Legacy anon, service_role API keys"

7️⃣ กรอกข้อมูลเข้าเว็บออกบิล

ขั้นตอนสุดท้าย นำค่าที่ได้จากขั้นตอนที่ 6 มากรอกในเว็บออกบิล

  1. เปิดเว็บออกบิล (index.html) จะเจอหน้า Login
  2. คลิก "⚙️ ตั้งค่าฐานข้อมูล Supabase ของฉัน (สำหรับผู้ใช้ใหม่)" — ปุ่มนี้อยู่ด้านล่างของหน้า Login ไม่ต้อง Login ก่อนก็กดได้
  3. วาง Supabase URL และ Anon Key — นำค่าที่ได้จากขั้นตอน 6A และ 6B มาวางในช่องที่ตรงกัน
  4. ตรวจสอบ URL ก่อนบันทึก — ย้ำอีกครั้ง: URL ต้องเป็น https://xxxxxxxxxxxxxxxx.supabase.co เท่านั้น — ไม่มี /rest/v1/ และไม่มี / ปิดท้าย
  5. กำหนดจำนวนห้องทั้งหมด — ใส่จำนวนห้องของหอพักคุณ (ค่าเริ่มต้น 10) ปรับเพิ่ม/ลดทีหลังได้ที่แท็บ "ตั้งค่าระบบ"
  6. กด "บันทึกและกลับไปเข้าสู่ระบบ" — ระบบจะพากลับไปหน้า Login อัตโนมัติ
  7. Login ด้วย Email/Password จากขั้นตอนที่ 5 — เข้าสู่ระบบสำเร็จ พร้อมใช้งานได้ทันที 🎉
✅ เสร็จแล้ว! ค่าที่กรอกจะถูกจดจำไว้ในเบราว์เซอร์นี้ ครั้งต่อไปเปิดเว็บมาจะ Login ได้เลยโดยไม่ต้องตั้งค่าซ้ำ
📱 เปลี่ยนเครื่อง/เบราว์เซอร์: ถ้าเปลี่ยนไปใช้เครื่องอื่น หรือเบราว์เซอร์อื่น จะต้องทำขั้นตอนนี้ใหม่ เพราะการตั้งค่าเก็บแยกตามเบราว์เซอร์

🔧 แก้ปัญหาที่พบบ่อย

ปัญหา สาเหตุที่เป็นไปได้ วิธีแก้
กดเข้าสู่ระบบแล้วไม่ขึ้นอะไร / ค้าง ยังไม่ได้ตั้งค่า Supabase URL/Key กดปุ่ม "⚙️ ตั้งค่าฐานข้อมูล" ที่หน้า Login แล้วกรอกใหม่ตามขั้นตอน 6-7
เข้าสู่ระบบไม่สำเร็จ: Invalid login credentials อีเมล/รหัสผ่านผิด หรือยังไม่ได้เปิด Auto Confirm User ตรวจสอบใน Supabase → Authentication → Users ว่าสร้างบัญชีไว้แล้ว และสถานะเป็น Confirmed
บันทึก/โหลดข้อมูลไม่ได้ (Failed to fetch) URL ผิดรูปแบบ เช่น มี /rest/v1/ ต่อท้าย หรือมี / ปิดท้าย ไปที่แท็บ "ตั้งค่าระบบ" แก้ URL ให้เหลือแค่ https://xxxx.supabase.co โดยไม่มี / ปิดท้าย
จำนวนห้องในเมนูไม่ตรงกับห้องจริง ยังไม่ได้ปรับค่า "จำนวนห้องทั้งหมดในหอพัก" ไปที่แท็บ "ตั้งค่าระบบ" แก้ตัวเลขจำนวนห้อง เมนูจะอัปเดตอัตโนมัติ
เปิดเว็บจากเครื่อง/เบราว์เซอร์อื่นแล้วต้องตั้งค่าใหม่ทุกครั้ง การตั้งค่าเก็บไว้ใน localStorage ของเบราว์เซอร์นั้นๆ เท่านั้น ทำตามขั้นตอนที่ 6-7 ซ้ำในเบราว์เซอร์ใหม่
หา Anon Key ไม่เจอ เข้าผิดเมนู หรือยังไม่ได้คลิกแท็บที่ถูกต้อง ไปที่ Project Settings › API Keys แล้วคลิกแท็บ "Legacy anon, service_role API keys" จะเห็นคีย์ anon ในนั้น

💬 ยังติดปัญหา? ถ้าทำตามทุกขั้นตอนแล้วยังติดปัญหา ลองใช้ขั้นตอนที่ 8 ตรวจสอบอัตโนมัติ เพื่อดูว่าปัญหาอยู่ตรงไหน หรือลองปิด-เปิดเบราว์เซอร์ใหม่ หรือลองเปิดเว็บในโหมดไม่ระบุตัวตน (Incognito) เพื่อตัดปัญหาเรื่องการตั้งค่าเก่าค้างอยู่
